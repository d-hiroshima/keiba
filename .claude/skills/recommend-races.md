# 推奨レースフロー（`/recommend`）

このスキルは **Agent Teams** を生成し、複数メンバーによる議論的な絞り込みで購入候補レース・買い目を提案する。**個別レース予想ではない**点が `/analyze` `/debate` との違い。

## 前提
- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 有効化済み
- Claude Code v2.1.32+
- 母集団: **当日開催の重賞 + プロンプトで指定したレース（race_id / keibalab URL）**
- 時間軸: **当日 / 週末**（直前情報を強く反映）
- データ鮮度: 出走表が古い場合は事前に `fetch_races.py` を実行

## チーム編成（5メンバー）

| Member | 名前 | 役割 |
|---|---|---|
| `macro-scout` | scout | 天候・馬場・直前情報の収集・要約 |
| `race-context-analyst` | context | レース選別、ペース・展開 |
| `pedigree-analyst` | pedigree | 候補各馬のかかりやすさ・血統スコア |
| `track-analyst` | track | コース適性・脚質・想定位置取り |
| `devils-advocate` | critic | 反証・穴レース・穴馬提示 |

## フェーズ別フロー

### フェーズ1: リサーチ（並列実行）

並列起動して独立タスクを実行：

- **scout**: 直前情報スナップショット作成（天候/馬場/調教/騎手/取消）
- **context**: 当日開催の全レースをコース・グレード別に分類し、**「妙味度」高いレース** をピックアップ（4プレイブックを最低1回ずつ確認）

リーダー（メインClaude）はこの間、母集団リスト（当日重賞 + プロンプトで指定されたレース）をまとめておく。

### フェーズ2: レース絞り込み（リーダーが実施）

scout と context の結果を集約し、母集団から以下条件で **5-8レース** に絞る：

1. **妙味度高** が望ましい（context が指摘したレース）
2. **馬場予想が立てやすい**（不確実性低い、または前日からの大変動なし）
3. **逃げ馬の頭数・脚質バランス** が読みやすい
4. **GI/GII を優先**、条件戦はプロンプトで明示指定された場合のみ対象
5. **レースリスト** を pedigree と track に SendMessage で渡す

### フェーズ3: 各レース深掘り（並列実行）

- **pedigree**: 各候補レースの上位人気馬 + 穴候補（合計15-30頭）に簡易血統スコア
- **track**: 各候補レースの上位人気馬 + 穴候補にコース適性スコア・脚質判定

両者は並列で動き、結果は共有タスクリストに掲載。

### フェーズ4: 反証ラウンド（critic 主導）

- **critic** が Top 3-5 候補レースに対して、各メンバーに **SendMessage で反論質問** を送る
- 反論観点:
  - 「このレースの本命は単勝オッズ過剰人気では」
  - 「想定ペースが外れたら差し決着で本命崩れでは」
  - 「直前で馬場が変わったら血統評価が逆転するのでは」
- 反論を受けてメンバーは応答・修正
- 議論は **最低1ラウンド**、必要なら2ラウンドで打ち切り
- **各レースに穴馬1-2頭** を必ず提示

### フェーズ5: 最終ランクと買い目策定（リーダー）

リーダーは全メンバーの最終スコアと議論サマリを統合：

- レースごとに本命・対抗・単穴・穴
- 買い目（ワイド・馬連・三連複・三連単）と推奨点数
- **Top 3-5 を確定**

### フェーズ6: 予算配分（オプション）

引数で `--budget=<円>` `--type=<券種>` `--points=<点数>` `--ev-threshold=<期待値>` が指定された場合：

- `docs/playbooks/wide-strategy.md` の手法に従って予算を配分
- 各レースの本命確信度 × 想定オッズ × 期待値で優先順位
- 期待値しきい値未満は除外
- 出力に **円単位の点数** を提示

### フェーズ7: チームクリーンアップ（必須）

「Clean up the team」をリーダーから指示。

## ガード

- **賭博助言ではない**: 末尾に必ず免責を付ける
- **データソースの鮮度** を明示（scout の取得時刻、出走表 DB の最新日）
- 議論で割れた点を **「合意」 で薄めない**: 議論済み論点として保持する
- 予算配分は **保守的** に推奨（最初は小、勝ち続けたら買い増し）
- 期待値しきい値（既定 1.0）未満は買わない

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| scout の WebFetch が失敗多発 | データソースを JRA 公式・netkeiba に限定 |
| 候補が3レースを下回る | 母集団を当日全重賞 + プロンプトで追加レースを指定してもらう |
| 議論が膠着 | リーダーから「2ラウンド目で打ち切り」と指示 |
| 最終出力が長すぎる | Top を3に絞る、または1レースあたりの自由記述を200字以内に |

## モード使い分け

| シーン | 推奨コマンド |
|---|---|
| 個別レースを一発で見たい | `/analyze` |
| GI を深く議論させたい | `/debate` |
| **今日／週末買うレースを提案させたい** | **`/recommend`** |
| 指定レース／馬のデータ最新化 | `/update-races <race_id>...` |
| 終了レースの採点・教訓抽出 | `/review-race <race_id>` |

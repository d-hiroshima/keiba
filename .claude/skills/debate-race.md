# 議論型レース分析（`/debate`）

このスキルは **Agent Teams** を使って 4 メンバーによる議論を行う。`/analyze` よりコスト高だが、**異なる視点間での反論・修正** が起きるため、判断のリスクが高いレースで使う。

## 前提
- `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` 有効化済み（`.claude/settings.json`）
- Claude Code v2.1.32+
- レース DB が古い場合は事前に `python scripts/fetch_races.py <race_id>` を推奨

## チーム編成（4メンバー）

| Member | 名前 | 役割 |
|---|---|---|
| `pedigree-analyst` | pedigree | 血統・産駒傾向 |
| `track-analyst` | track | コース・距離・脚質 |
| `race-context-analyst` | context | レースレベル・ペース・展開 |
| `devils-advocate` | critic | 反証・穴馬提示 |

## フェーズ別フロー

### フェーズ1: 事前準備
1. `race_id` を正規化、`races` テーブルでコース・距離・グレード取得
2. データ鮮度確認
3. `<course>-<distance>` プレイブックの存在確認

### フェーズ2: チーム生成

リーダーが自然言語でチーム生成依頼：

> 「`<race_id>`（<course> <distance>m <grade>）について議論型のレース予想を行うエージェントチームを作成してください。次の subagent 定義を使って4チームメンバーを生成：
>
> - `pedigree-analyst` 名: `pedigree`
> - `track-analyst` 名: `track`
> - `race-context-analyst` 名: `context`
> - `devils-advocate` 名: `critic`
>
> 各メンバーは初期分析を提示した後、`critic` が他メンバーに SendMessage で反論質問を送ります。各メンバーは反論または部分修正で応答し、最低1ラウンド議論を行います。`context` は `docs/playbooks/<course>-<distance>.md` を必読です。」

### フェーズ3: 議論進行のモニタリング

リーダーは以下を監督：
- 全メンバーが初期分析を提示したか
- `critic` が各メンバーに反論質問を送ったか
- 議論が **同じ論点を繰り返している** 場合 → 「次の論点に進んで」と介入
- 議論が **収束した** 場合 → 「最終結論をまとめて」と指示
- 議論が **メイン論点から外れた** 場合 → 「論点を <X> に戻して」と介入

### フェーズ4: 統合判断

各メンバーの最終結論を集約して、以下のフォーマットで出力：

```
# 議論型レース予想: <race_id> <race_name>（<course> <distance>m <grade>）
日付: YYYY-MM-DD（発走 HH:MM）
モード: /debate（Agent Teams）

## 議論の主要争点
1. <争点1>: <pedigree の主張> vs <critic の反論> → <結論>
2. <争点2>: ...
3. <争点3>: ...

## メンバー別最終結論
| Member | 強気度 | 確信度 | 議論を経ての変化 |
|---|---|---|---|
| pedigree | | | 例: 当初+3 → 議論後+2（産駒N不足の指摘で確信度低下） |
| track | | | |
| context | | | |
| critic | | | |

## 統合判断
- **本命（◎）**: <horse_name>
- **対抗（○）**: <horse_name>
- **単穴（▲）**: <horse_name>
- **穴（▽）**: <horse_name>（critic 推奨）
- **議論を経て明確になった点**: ...
- **議論しても合意に至らなかった点**: ... ← **買い目におけるリスク**

## 買い目案
- ワイド: ...
- 馬連: ...
- 三連複: ...
- 推奨予算配分: 軸 X% / 押さえ Y% / 穴 Z%

## 反証条件（合意できなかった点が顕在化したら判断を変える）
- ...

## 次に観察すべきデータ
- 当日の馬場発表（X時X分予定）
- パドック気配
```

### フェーズ5: チームクリーンアップ（必須）

分析完了後、リーダーから「Clean up the team」と指示してチームを解散。

> **重要**: 解散しないとセッション切替後にゾンビ状態のチームメンバーが残る場合あり（Agent Teams の既知の制限）。

## モード使い分けの目安

| 状況 | 推奨コマンド |
|---|---|
| ウォッチ重賞の事前確認 | `/analyze --quick` |
| 個別レースの通常予想 | `/analyze` |
| **GI レースの本命検討** | **`/debate`** |
| **本命と対抗で意見が割れそうな複雑レース** | **`/debate`** |

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| `/agents` でチームメンバーが見えない | Shift+Down でサイクル、または分割ペインモードに切替 |
| 議論が止まる | リーダーから「次の論点に進んで」と直接介入 |
| トークン消費が異常に多い | チームメンバー数を3に減らす、または `/analyze` に切替 |
| 議論終了後もチームが残る | `Clean up the team` を必ず実行 |

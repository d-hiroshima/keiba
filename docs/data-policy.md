# データ分類ポリシー（何を git にコミットしてよいか）

keibalab のデータは **個人利用・非再配布** が前提（CLAUDE.md「データ取得」）。
このリポジトリを公開リポジトリとして扱う場合の分類を以下に固定する。
「公開してよいか迷うもの」はコミットしない側に倒す。

## コミットして良いもの

| 種別 | 例 | 理由 |
|---|---|---|
| コード・スキーマ | `scripts/`, `docs/db-schema.md` | 自作物 |
| プロンプト・手順・プレイブック | `.claude/`, `docs/playbooks/` | 自作物（出典明記の傾向値は引用の範囲） |
| **合成フィクスチャ** | `tests/fixtures/*.html` | 実ページの**構造だけ**を真似た架空データ（実在の馬名・実数値を含めない） |
| 予想成果物・採点・postmortem | `predictions/`, `docs/postmortems/` | 自作の分析。ただし下記の注意 |

## コミットしてはいけないもの

| 種別 | 例 | 理由 |
|---|---|---|
| スクレイプした生 HTML | `data/cache/` | keibalab コンテンツの再配布になる（gitignore 済み） |
| DB 本体・一括ダンプ | `data/race.db`, `data/*.json` | 同上（gitignore 済み） |
| keibalab 独自指標 | α/β/Ω指数 | 同社 IP。**DB にも入れない** |
| 認証情報 | `.env`, トークン | 言わずもがな（gitignore 済み） |

## 注意（予想成果物）

- 予想 md には keibalab 由来の事実（着順・オッズ等）が**分析に必要な範囲で**引用される。
  これは自作分析の一部として許容するが、**戦績表の丸写し・一括転載はしない**。
- 買い目の金額は個人の資金情報。公開が気になる場合は金額を伏せるか、
  リポジトリを private にする（推奨: private 運用）。
- watchlist を廃止した経緯（追跡対象を公開リポジトリに残さない）と同じ判断基準を適用する。

## テストフィクスチャの作り方

実ページをコミットせず、構造（タグ・クラス名・カラム構成）を保った **架空の最小 HTML** を手書きする。
馬名・騎手名・数値はすべて架空とする。実ページとの構造差分が疑われる場合は
`data/cache/`（ローカル）と目視比較し、フィクスチャ側を更新する。

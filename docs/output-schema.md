# 出力スキーマ（共通）

このプロジェクト（A: Claude Code / B: Slack bot）の **すべてのエージェント・スキル出力** が準拠する規約。**JSON ではなく Markdown** で、必須セクションと必須カラムだけを規定する。装飾・自由記述・順序の入れ替えは認める。

## 1. 設計方針

- **Markdown を正**: 人間可読を最優先。Slack bot 側のレンダラーがパースする。
- **必須セクションは漏れたらエラー扱い**: 統合時にメインClaude が検証する。
- **任意セクションは自由**: レース固有の論点（淀の下り、Cコース替わり等）は自由記述で残せる。
- **カラム順序は固定**: 全頭評価表など比較対象になる表は順序を揃えると振り返りが楽。
- **A と B で共通**: `slack-bot/prompts/load_agent_prompt.py` で同じ `.claude/agents/*.md` を読むため、スキーマも同一。

## 2. 共通ヘッダ（すべての出力の冒頭）

エージェント単体出力・統合出力ともに、必ず以下を1行目〜数行目に含める。

```
## <agent-name or skill-name> — race: <race_id>
取得日時: YYYY-MM-DD HH:MM JST
モード: <subagent | team | recommend>  # 任意（複数モード対応エージェントのみ）
```

`race_id` は12桁。複数レース対応の `/recommend` では `races: [<id1>, <id2>, ...]` でも可。

**取得日時は幻覚で書かない（時刻の真実源）**: 出力を書き始める前に必ず
`TZ=Asia/Tokyo date '+%Y-%m-%d %H:%M JST'` を Bash で実行し、その値を転記する。
発走までの残り時間に言及する場合は、`races.post_time`（DB）または取得した発走時刻との差で計算する。
現在時刻・発走時刻のどちらかが取れない場合は「残り時間: 不明」と書く（推定で埋めない）。

## 3. エージェント別の必須セクション

### 3.1 pedigree-analyst（血統）

| セクション | 必須 | 内容 |
|---|---|---|
| `### 全頭評価サマリ（出走全頭）` | ✓ | 表（必須カラム下記） |
| `### 詳細評価` | ✓ | 強気度上位 5-7 頭＋消し上位 1-2 頭 |
| `### 産駒の同条件成績` | △ | データ不足時は省略可・確信度で減点 |
| `### 自由記述` | ✓ | 200字以内 |

**全頭評価サマリ表 必須カラム（順序固定）**:
`馬番 | 馬名 | 父 | 母父 | 同条件父系勝率 | 強気度 | 確信度 | キー論点（30字以内）`

### 3.2 track-analyst（コース・脚質・騎手）

| セクション | 必須 | 内容 |
|---|---|---|
| `### 全頭評価サマリ（出走全頭）` | ✓ | 表 |
| `### 想定ペース全体像` | ✓ | Hi/M/Sl と上がり要求、隊列予想 |
| `### 詳細評価` | ✓ | 上位5-7頭＋消し1-2頭 |

**全頭評価サマリ表 必須カラム（順序固定）**:
`馬番 | 馬名 | 同条件勝率 | 持ち時計 | 想定脚質 | 想定位置取り | コース適性 | 騎手 | 騎手当該コース勝率 | コンビ/乗替 | 強気度 | 確信度 | キー論点`

### 3.3 race-context-analyst（展開・レベル）

| セクション | 必須 | 内容 |
|---|---|---|
| 冒頭ブロック（強気度・確信度・レースレベル判定・想定ペース） | ✓ | 4項目固定 |
| `### 全頭の展開恩恵度（出走全頭）` | ✓ | 表 |
| `**有利不利マトリクス**` | ✓ | 脚質別 ◎○△▲× |
| `**観察すべき次のデータポイント**` | ✓ | 1-3点 |

**展開恩恵度表 必須カラム（順序固定）**:
`馬番 | 馬名 | 想定脚質 | 想定位置取り | 展開恩恵度 | 強気度 | 確信度 | キー論点`

### 3.4 macro-scout（揮発データ）

| セクション | 必須 | 内容 |
|---|---|---|
| 冒頭の取得日時・発走までの残り時間 | ✓ | 必ず記載 |
| `### 天候・馬場予想` | ✓ | 4-5項目 |
| `### 当日の馬場発表` | ✓ | 取得失敗時は「取得失敗」と明示 |
| `### オッズ` | ✓ | 表（必須カラム下記） |
| `### 馬体重・気配（当日値）` | ✓ | 全頭または取得分 |
| `### 周辺重賞・前走` | ✓ | 主要トライアル組 |
| `### 判定の前提となる重要トリガー` | ✓ | 1-5点 |

**オッズ表 必須カラム（順序固定）**:
`馬番 | 馬名 | 単勝 | 人気 | 前日比`

主要馬の **複勝レンジ・ワイドオッズ・馬連オッズ** は表外で別途記載してよい（買い目策定で参照するため）。

### 3.5 devils-advocate（反証）

| セクション | 必須 | 内容 |
|---|---|---|
| `### 本命論の要約（公平に）` | ✓ | 100-200字 |
| `### 全頭評価サマリ横断レビュー` | ✓ | 評価割れ・過小・過剰の3観点表 |
| `### 弱点（最低3点）` | ✓ | 番号付き |
| `### 「外れるシナリオ」と穴馬候補` | ✓ | 最低2シナリオ |
| `### データソースの限界` | ✓ | 1-3点 |
| `### 全頭への補正提案` | ✓ | 表（出走全頭、+/- 0 でもよい） |
| `### 結論` | ✓ | 強気度・確信度・反証条件・自由記述（穴馬2頭以上） |

**全頭への補正提案 必須カラム（順序固定）**:
`馬番 | 馬名 | 他エージェント総合 | devils 補正 | 補正理由`

## 4. 統合出力（メインClaude / `/analyze` ステップ4）

**必須セクション一覧（10項目）**:

1. `# レース予想: <race_id> <race_name>（<course> <distance>m <surface>, <grade>）` ヘッダ行
2. `日付: YYYY-MM-DD（発走 HH:MM） / 出走 N 頭` メタ行
3. `## データ取得状況` — 出馬表取得状況・揮発データ取得時刻
4. `## エージェント別結論サマリ` — エージェント × 強気度・確信度・キー論点
5. `## 全頭評価サマリ（必須・出走全頭）` — 全頭横断統合表
6. `## 統合判断` — 本命/対抗/単穴/押さえ/穴
7. `## 想定ペース・展開` — ペース・有利脚質・馬場想定
8. `## 買い目（参考）` — 券種別 ※ユーザーが予算指定した場合は配分明示
9. `## リスクと反証条件` — メインリスク3点＋トリガー＋直前確認
10. `## 次に観察すべきデータ` — 1-5点

**全頭評価サマリ表 必須カラム（順序固定）**:
`馬番 | 馬名 | 想定人気 | pedigree | track | race-context | devils | 総合 | 選定 | キー論点`

- `想定人気`: macro-scout の単勝オッズ順位（取得時刻も明示）
- `pedigree / track / race-context / devils`: 各エージェントの強気度（-5〜+5）
- `総合`: メインClaude の加重統合（-5〜+5、平均値ベースだが評論家の弱気は薄めない）
- `選定`: `◎ / ○ / ▲ / △ / ▽ / -`

**買い目セクションのフォーマット**:

ユーザー予算指定時（例: `ワイド2000円・三連複2000円`）は、必ず以下を含める:

```
### <券種> <予算>円
| 買い目 | 推定オッズ | 投資額 | 想定払戻 |
|---|---|---|---|
| ... | ... | XXX 円 | ... 円 |
| **合計** | | <total> 円 | |
```

合計が予算と一致することをユーザーが目視確認できるようにする。

## 5. 必須フィールドの型・値域

| フィールド | 型 | 値域 |
|---|---|---|
| 強気度 | int | -5 〜 +5 |
| 確信度 | int | 1 〜 5 |
| 想定脚質 | enum | 逃げ / 先行 / 差し / 追込 |
| 想定ペース | enum | Hi / M / Sl |
| 適性記号 | enum | ◎ / ○ / △ / ▲ / × |
| 選定 | enum | ◎ / ○ / ▲ / △ / ▽ / - |
| キー論点 | str | 30字以内（表内セル） |

## 6. ガード（共通）

- **必須セクション欠落 = エラー**: メインClaude は統合時に検証し、欠けていれば該当エージェントへ再依頼
- **「全頭評価」は出馬表全頭が網羅されていること**: 1頭でも欠けたら未完成
- **データ取得失敗の明示**: `「取得失敗」`/`「取得不可」`/`「N=不明」` のいずれかで明記。空欄禁止
- **取得時刻と発走時刻の明示**: 揮発データを引用するセクションには必ず取得時刻（時刻は §2 の手順で実測する）
- **賭博助言ではない注記**: 統合出力の末尾に1行
- **出典 URL**: 外部参照したセクションには必ず付ける（DB から取った場合は `data/race.db` と明記）

### N=0 縮退規約（データ欠損時の必須手順）

「同条件父系勝率」「騎手当該コース勝率」「同条件勝率」など **DB 集計を前提とする数値カラム** で、
ローカル DB の該当サンプルが無い／少ない場合は、**勝率をでっち上げず** 次の順で縮退する：

1. **N=0 を明記**: セルには `N=0` と書く（例: `-(N=0)`）。空欄・捏造数値は禁止
2. **WebFetch フォールバック（権限がある場合のみ）**: pedigree-analyst は keibalab/netkeiba の
   種牡馬ページ等で補完してよい。ただし同条件クロス表は外部に無いため、取得値は
   `総合XX%(近似・出典URL)` と **近似であることを明記**。WebFetch を持たないエージェント
   （track-analyst）は macro-scout かメインClaude に依頼するか、縮退のまま進める
3. **LLM 一般知識で補う場合**: `出典: LLM一般知識・要検証` と明記し、その馬の **確信度上限は 2**
4. **N が小さい場合**（1≤N<10 など）: 数値は書いてよいが `XX%(N=X, 小標本)` と明記し、
   強気度の根拠に使わない（track-analyst の N 閾値ルールと整合）

`pedigree_stats.n_horses` が 1 の行は **1頭の個体成績の言い換え** なので「産駒傾向」として扱わない。

## 7. Slack bot 側の期待（B 形態）

`slack-bot/orchestrator.py` で各エージェント結果を辞書化する際、以下のキーを抽出する想定。

| キー | 抽出元 | 用途 |
|---|---|---|
| `agent_name` | 共通ヘッダ | カード見出し |
| `race_id` | 共通ヘッダ | スレッド紐付け |
| `summary_table` | 全頭評価サマリ表 | Slack canvas のテーブル描画 |
| `verdict` | 詳細評価／結論 | カード本文 |
| `confidence` | 結論／冒頭 | バッジ表示 |
| `betting_plan` | 買い目セクション | 別カード |

スキーマ違反は orchestrator が検出してログに `SCHEMA_VIOLATION` を残す（Phase 3 で実装）。

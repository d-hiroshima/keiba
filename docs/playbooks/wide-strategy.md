# ワイド予算戦略プレイブック

> `--budget` `--type` `--points` `--ev-threshold` を受け取って買い目と点数を最適化する方法論。
> A・B 共用を想定。Phase 4 で共通モジュール化する。

## 1. 概要

競馬の買い目戦略は **券種選択 × 軸馬選定 × 相手選定 × 予算配分** の 4 軸で決まる。本プレイブックは「期待値ベース」での予算最適化フレームを示す。

> **注意**: 期待値が 1.0 を超えると数学的には買う方が得だが、的中までの分散が大きいため、**長期で続けて初めて回収率に収束**する。短期の的中・不的中で評価しないこと。

## 2. 券種別の妙味比較

| 券種 | 期待値出しやすさ | 分散 | 推奨用途 |
|---|---|---|---|
| 単勝 | △（控除率 20%） | 中 | 軸馬への確信度高い時 |
| 複勝 | △（控除率 20%、オッズ低い） | 低 | 安定収益を狙う時 |
| ワイド | ○（3 着までで的中） | 中 | **本命党のメイン券種** |
| 馬連 | ○（控除率 22.5%） | 中-高 | 軸 - 相手の組合せ妙味 |
| 馬単 | △（控除率 22.5%、点数嵩む） | 高 | 軸馬の勝ちパターン明確時 |
| 三連複 | ◎（妙味出しやすい） | 中-高 | **穴を絡めた時の妙味券種** |
| 三連単 | △（控除率 27.5%、点数嵩む） | 非常に高 | 配当狙いの少額勝負 |

**推奨**: 軸 1 頭流しのワイド or 三連複が、本命型予想と相性が良い。

## 3. 期待値計算の基本フレーム

### 3-1. 想定的中確率の算出

各馬の **想定勝率** を 5 段階で評価：
- ◎ 本命（想定勝率 25-35%）
- ○ 対抗（15-25%）
- ▲ 単穴（10-15%）
- △ 押さえ（5-10%）
- ▽ 穴（3-5%）

各馬の **想定連対率（2着以内）** は、勝率 × 1.7 〜 1.8 倍が目安。
**想定複勝率（3着以内）** は、勝率 × 2.4 〜 2.6 倍。

### 3-2. 期待値の計算

```
期待値 = 想定的中確率 × オッズ
```

**期待値 1.2 以上を「買い妙味あり」と判定**（控除率を考慮するとこのライン以上が長期回収率プラスの目安）。

### 3-3. ワイド・三連複の組合せ計算

ワイド `A — B` の的中確率 ≒ A の複勝率 × B の複勝率（独立性の近似）
三連複 `A — B — C` の的中確率 ≒ 上記の拡張

> **正確には**：A と B の同時的中は独立ではない（同じレースなので）。実務では **JRA 公式の理論オッズと実際オッズの乖離** を見るのが現実的。

## 4. 予算配分の方法論

### 4-1. ベースライン: 確信度配分
本命確信度に応じて予算を傾斜配分。

| 軸の確信度 | 軸馬中心配分 | 例（予算 3000円） |
|---|---|---|
| 5（鉄板） | 軸 60% / 相手 30% / 穴 10% | ワイド ◎-○▲ 1800円 + ◎-△ 600円 + ◎-▽ 300円 + 三連複 300円 |
| 4 | 軸 50% / 相手 35% / 穴 15% | ワイド主軸を分散 |
| 3 | 軸 40% / 相手 40% / 穴 20% | 軸固めしすぎない |
| 2 以下 | レース見送り or 少額のお試し | 1000円以内 |

### 4-2. 期待値配分（上級）
各組合せの期待値 × 想定的中確率 で予算を傾斜配分。
詳細は Phase 4 で実装する Python モジュール（`scripts/wide_optimizer.py` 仮）に組み込む予定。

擬似コード：
```python
def optimize_budget(candidates, budget, ev_threshold=1.2, max_points=8):
    # candidates: [{combo, expected_odds, prob, ev}, ...]
    candidates = [c for c in candidates if c["ev"] >= ev_threshold]
    candidates = sorted(candidates, key=lambda c: c["ev"], reverse=True)[:max_points]
    total_ev = sum(c["ev"] * c["prob"] for c in candidates)
    return [
        {**c, "amount": int(budget * (c["ev"] * c["prob"]) / total_ev / 100) * 100}
        for c in candidates
    ]
```

### 4-3. 最低単位
JRA は **100 円単位**。点数を増やしすぎて 100 円割れになる場合は、点数を絞るか予算を上げる。

## 5. 引数仕様

`/recommend --budget=3000 --type=wide --points=6 --ev-threshold=1.2` の解釈：

- `--budget`: 総予算（円）
- `--type`: 券種（`win` / `place` / `wide` / `umaren` / `umatan` / `sanrenpuku` / `sanrentan`）。複数指定 `--type=wide,sanrenpuku` も検討
- `--points`: 点数上限。指定なしなら期待値順で自動決定
- `--ev-threshold`: 期待値しきい値（既定 1.2）

## 6. 買い目テンプレート

### 6-1. 軸 1 頭流し（本命確信度高い時）

ワイド: ◎ — ○▲△ （3 点、配分は本命相手の信頼度で傾斜）
三連複: ◎ — ○▲△ — ○▲△▽ （6-9 点）

### 6-2. 軸 2 頭ボックス（軸が並立）

ワイド: ◎○ ボックス（1 点、配分大きめ）
三連複: ◎○ — ▲△▽ — ▲△▽（複数点）

### 6-3. 穴狙いミックス

ワイド: ◎ — ○▲△ + ▽ — ○▲ （穴のワイドを軽め）
三連複: ◎ — ▲△ — ▽ （穴を含めた組み合わせ）

## 8. 守るべきガード

- **賭博助言ではない**: 出力は予想と買い目の参考情報。最終判断はユーザー責任
- **資金管理**: 1 レースあたりの予算は **総資金の 1-3%** を超えない（プロでも 5% 超は危険）
- **追い掛け禁止**: 負けたレースの直後に予算を増やすのは破滅パターン
- **連敗期間の認識**: 期待値 1.2 でも 10 レース連続外しもあり得る分散
- **券種ミックス**: 1 レースに全予算を 1 券種に投じない（保険として 2-3 券種）
